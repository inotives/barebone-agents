# Preference body

Free-form markdown explaining the preference in detail. The body is what
the agent sees in its system prompt when this preference is selected.

The selector matches preferences against the task title + description (or
the first user message of a conversation segment). Add `keywords` for the
most predictable terms; the body content also contributes to matching.

## Scope

Set `scope:` to one of:

- `global` — always inject (use for personal style rules that should apply
  to every task).
- `<topic>` — domain-specific (e.g. `git`, `backend-rust`, `research-finance`).
  Inject only when keywords match.

## Notes

- This file (`.template.md`) is skipped by the loader (dot-prefix sentinel).
  Copy it to `<your-slug>.md` to author a real preference.
- Local edits to existing preferences are auto-pushed to AKW by the
  background pusher (default hourly). The manifest at
  `data/.akw_push_manifest.json` tracks what's already been backed up.
- Reflection-generated preferences land in
  `data/drafts/2_knowledges/preferences/` (review-gated). Promote with
  `barebone-agent prefs promote <slug>`.
